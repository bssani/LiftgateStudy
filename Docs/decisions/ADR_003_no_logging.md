# ADR-003 — No Evaluation Data Logging

**Date**: 2026-05-13
**Status**: Accepted
**Phase**: 1

## Context

평가 시 정량 데이터 (liftgate 각도, 개폐 시간, zone hit, grab 위치, 평가자별 trial 번호 등) 로깅 여부 결정 필요.

로깅하면 사후 분석과 평가자 간 비교가 가능해지지만:

- 데이터 schema 협의 / 검증에 시간 소요
- 평가자 개인정보 처리 부담
- 평가자가 "기록된다"는 인식이 자연스러운 인터랙션 방해 가능성
- 본 도구는 "주관적 느낌" 평가가 핵심 가치

## Decision

**일체 로깅하지 않는다.** 평가는 평가자의 주관적 체험에만 의존한다. CSV / JSON / DB / 파일 출력 코드 작성을 금지한다.

CLAUDE.md §3 R7에 박제됨.

## Consequences

**Positive**:
- 구현 단순화 (로깅 인프라 불필요)
- 평가자 부담 감소 (개인정보 / 평가내용 익명성)
- 빠른 iteration (data schema 협의 불필요)
- 평가자가 자연스럽게 인터랙션에 몰입

**Negative**:
- 사후 정량 분석 불가
- 평가자 간 정량 비교 불가
- 시간 경과 후 평가 결과 재현 / 검증 불가
- 동일 차량 / mode에 대해 여러 평가자 결과 합산 불가

**Neutral**:
- 평가자가 직접 메모 / 코멘트는 외부 (종이, 외부 툴) 에서 자유롭게

## Notes

- 향후 정량 평가가 필요해지면 별도 Phase로 분리하여 도입
- 본 프로젝트는 "주관적 느낌 전달" 도구로 고정
- 디버그 로그 (`UE_LOG`, on-screen debug) 는 개발 중에만 사용 가능. 빌드 산출물에는 평가 데이터 출력 일체 금지.
